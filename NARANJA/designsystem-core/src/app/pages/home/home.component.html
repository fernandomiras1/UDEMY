Zumo Front Library

V 2.6.0 publish

Zumo es una libreria que suministra un set de componentes de Angular, para ser reutilizado en varios proyectos de Naranja.

Instalacion

Zumo está disponible en un npm privado de Naranja. Antes de instalar esta librería, es necesario estar seguro de tener acceso al repositorio.

Agregar archivo .npmrc

Agregar este archivo permite ingresar al servidor npm privado de Naranja con el fin de poder descargar la librería. El mismo se agrega en la misma jerarquía que package.json. Con esta configuración, en el pipeline donde se instalan las dependencias se descargará Zumo automáticamente.

```bash registry=http://10.10.90.191:8081/repository/nx-npm-group/ init.author.name=Pipeline init.author.email=pipeline@naranja.com init.author.url=http://blog.example.com email=pipeline@naranaja.com always-auth=true _auth="bnBtLXp1bW86Tnp1bW9ucG0=" ```

Importar la librería en un proyecto:

```node npm install --save zumo ```

Agregar configuraciones a angular.json

Dentro del objeto json architect, en el array assets agregar:

```json { "glob": "**/*", "input": "node_modules/zumo/assets/fonts/", "output": "./assets/fonts/" } ```

Agregar configuraciones a tsconfig.json

Agregar el objeto json, al final del archivo.

```json "paths": { "zumo/*": ["node_modules/zumo"] } ```

Incluir la librería en un proyecto:

```typescript import { NgModule } from '@angular/core'; // asi se importa la libreria import { ZumoModule } from 'zumo'; @NgModule({ imports: [ // otros imports ZumoModule ] }) export class SharedModule { } ```

Incluir estilos

Para incluir los estilos de Zumo, agregar en el style.scss:

```sass // importar estilos @import "~zumo/vendor/reset"; @import "~zumo/colors"; @import "~zumo/variables"; @import "~zumo/typography"; @import "~zumo/spaces"; @import "~zumo/iconsFont"; @import "~zumo/html5"; @import "~zumo/animations"; @import "~zumo/vendor/flexboxgrid/flexboxgrid"; ```

Polyfills

Para tener compatibilidad con algunos navegadores (como Internet Explorer 9, 10 u 11) hay que configurar el archivo polyfills.ts. Conocer más.

```typescript /** IE9, IE10 and IE11 requires all of the following polyfills. **/ import 'core-js/es6/symbol'; import 'core-js/es6/object'; import 'core-js/es6/function'; import 'core-js/es6/parse-int'; import 'core-js/es6/parse-float'; import 'core-js/es6/number'; import 'core-js/es6/math'; import 'core-js/es6/string'; import 'core-js/es6/date'; import 'core-js/es6/array'; import 'core-js/es6/regexp'; import 'core-js/es6/map'; import 'core-js/es6/weak-map'; import 'core-js/es6/set'; /** IE10 and IE11 requires the following for NgClass support on SVG elements */ import 'classlist.js'; // Run `npm install --save classlist.js`. /** IE10 and IE11 requires the following for the Reflect API. */ import 'core-js/es6/reflect'; /** Evergreen browsers require these. **/ // Used for reflect-metadata in JIT. If you use AOT (and only Angular decorators), you can remove. import 'core-js/es7/reflect'; /** * Web Animations `@angular/platform-browser/animations` * Only required if AnimationBuilder is used within the application and using IE/Edge or Safari. * Standard animation support in Angular DOES NOT require any polyfills (as of Angular 6.0). **/ import 'web-animations-js'; // Run `npm install --save web-animations-js`. ```

Estado de Componentes

El componente es estable, tiene soporte total y está listo para ser usado en productos.

El componente está en desarrollo, en una versión estable pero no con todas sus funcionalidades.

El componente ya no tiene mantenimiento ni se actualizará.
